\## サイト概要（暫定）



\- \*\*形式:\*\* HTML + CSS + JavaScript（フレームワークなし・静的サイト）

\- \*\*コンセプト:\*\* RPG のコマンド選択 UI でページを切り替える就活用ポートフォリオ

\- \*\*主ゴール:\*\* 採用担当者に「ゲームエンジニア志望」「人間味」「制作姿勢」が伝わるサイトを公開する



\### 1 回の訪問の流れ（目標 UX）



1\. \*\*TOP（ホーム）\*\* — 背景 + 床 + キャラクターが表示される

2\. \*\*コマンドメニュー\*\* — 吹き出し風パネルに `About` / `Skills` / `Works` / `Contact` などが並ぶ

3\. \*\*コマンド選択\*\* — 選んだ項目の詳細画面に切り替わる（コマンドは \*\*非表示\*\*）

4\. \*\*TOP に戻る\*\* — 「戻る」操作で TOP に戻り、コマンドを \*\*再表示\*\*



\### 画面状態（設計の中心）



| 状態 ID | 画面 | コマンド表示 | 備考 |

|---------|------|-------------|------|

| `top` | TOP（キャラ＋背景） | ✅ 表示 | 初期画面 |

| `about` | 自己紹介 | ❌ 非表示 | |

| `skills` | スキル | ❌ 非表示 | |

| `works` | 制作実績 | ❌ 非表示 | |

| `contact` | 連絡先 | ❌ 非表示 | |



※ 追加候補（未決定）: `timeline` / `github`



\### 技術方針（初心者向け）



\- \*\*1 ページ構成を推奨\*\* — `index.html` 1 枚 + JavaScript で画面切り替え（`display: none` または CSS クラス）

\- \*\*状態は 1 変数で管理\*\* — 例: `let currentScreen = 'top'`

\- \*\*画像は CSS の `background-image` または `<img>`\*\* — 背景・床・キャラはすでに用意済みの素材を使用

\- \*\*参考実装:\*\* `AIMade\_Takahashi\_PortfolioSite/takahashi\_portfolio\_site/`（配色・文言・カード UI の参考。RPG 画面切り替えは本プロジェクトで新規実装）



\### やらないこと（初期版）



\- React / Vue などのフレームワーク導入

\- バックエンド・データベース

\- 複雑なアニメーションエンジン

\- ゲーム本編のような移動・戦闘システム
